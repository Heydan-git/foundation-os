2 Contexte — il n'est pas gratuit

Le contexte (input window) a une taille limitee. Plus tu charges de fichiers, plus je perds en precision sur les details.

Seuils observables

  • 0-40% precision haute, je cite ligne par ligne
  • 40-60% precision correcte, premieres compressions silencieuses
  • 60-80% hallucinations sur paths/lignes augmentent
  • 80%+ compactage automatique imminent, perte de nuances

Mitigations

  • Lire en parallele dans un seul message (meme cout, gain wall-clock)
  • Deleguer les readings exploratoires a un sub-agent (contexte isole)
  • Decouper en phases A-F au lieu d'un monolithe (pattern Cycle 3)
  • Declarer un budget de fichiers en debut de session
Reflexe : avant de charger un 8e fichier, demande-toi si tu peux fermer 3 anciens.

6 Skills et MCPs — outils a comprendre, pas a multiplier

Skills

Un skill ajoute des instructions specialisees au debut de la session. Cout : tokens en input. Gain : qualite de l'output dans le domaine.

  • Skills Foundation OS : foundation-os-orchestrator (a reecrire), fullstack-dev, database-architect, ui-expert, lead-dev
  • Skills domaine perso : health-council, nutritionniste, crypto-trader, etc.
  • Meta-skills : skill-creator, skill-supervisor. A utiliser avec moderation.

MCPs

Un MCP donne acces a un service externe (Asana, Notion, Figma, Chrome, computer-use). Cout : appels reseau + tokens schema. Gain : actions reelles hors sandbox.

  • MCP dedie > Chrome MCP > computer-use (par ordre de fiabilite)
  • computer-use est OK pour apps natives (Notes, Finder, Mail) mais lent et fragile
  • Aucun MCP GitHub ni MCP Supabase actuellement → potentiel d'amelioration
Reflexe : avant d'activer un nouveau skill, se demander "ce skill va-t-il etre utile dans > 20% de mes sessions ?". Sinon c'est noise.

7 Workflows — patterns qui marchent

Pattern 6 phases A-F (anti-compactage)

  • A — Read parallele (ingestion materiau)
  • B — Audit ciblé
  • C — Verification croisee
  • D — Findings + decisions
  • E — Livrable
  • F — Consolidation + commit (peut etre en session 2 si phase A-E deja committee WIP)

Niveaux PAUL pour /session-end

  • DONE tout est verifie, build OK, tests OK, commit prêt
  • DONE_WITH_CONCERNS termine mais avec dette identifiee
  • NEEDS_CONTEXT il manque une info Kevin pour finir
  • BLOCKED impossible de continuer sans intervention externe

MD-first / runtime-then-doc

Quand un script et sa doc divergent, le RUNTIME (script) fait foi. Aligner la doc dessus, pas l'inverse.

Reflexe : ne jamais finir une session sans /session-end (meme rapide). Le rituel est ce qui maintient CONTEXT.md vivant.

8 Garde-fous anti-bullshit

Mots interdits (CLAUDE.md)

  • × revolution
  • × historique
  • × reference mondiale
  • × premier au monde
  • × world-first / revolutionary / accomplish

Regles de verification

  • Jamais "TERMINE" ou "100%" sans build + test executes
  • Chaque metrique doit avoir une commande de verification
  • Plus de MD que de code dans une session = red flag
  • Un fichier qui documente un autre fichier = du bloat
