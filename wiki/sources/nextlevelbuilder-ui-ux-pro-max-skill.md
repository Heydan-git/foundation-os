# UI UX Pro Max — skill design intelligence 67 styles + 161 palettes + 57 fonts

## Summary

**Claude Code skill mature** (67.7k stars, MIT, v2.5.0 mars 2026) par nextlevelbuilder. Fournit un **Design System Generator reasoning engine** qui matches product type user → recommande styles + color palettes + font pairings + UX patterns automatiquement via BM25 ranking.

**Base de donnees massive** : 67 styles / 161 palettes / 57 fonts / 25 chart types / 99 UX guidelines / 161 reasoning rules / 24 landing page patterns. Format CSV + Python scripts.

**Compat 18 AI assistants** (Claude Code, Cursor, Windsurf, Antigravity, Copilot, Gemini CLI, Codex, etc.) et **14+ frameworks** (React/Next.js/shadcn/Vue/Nuxt/Svelte/Astro/Angular/Laravel/SwiftUI/Jetpack/RN/Flutter/HTML+Tailwind).

**Verdict FOS** : **consultation reference SEULEMENT, ne PAS installer globalement** — conflict direct avec contrainte [[Void Glass]] dark-only FOS. Details FOS Analysis ci-dessous.

## Foundation OS Analysis

### Compat OS — CRITIQUE

**Si installe globalement ~/.claude/skills/ = LOW**.

**Raison fondamentale** : le skill auto-active sur keywords UI/UX et **suggerera des styles divers** (161 palettes, 57 fonts, 67 styles). **FOS a [[Void Glass]]** — contrainte **dark-only** specifique avec :
- Surface `#030303` (`ds-surface-0`)
- Tokens CSS variables `--ds-*` uniquement
- Fonts **Figtree** (UI) + **JetBrains Mono** (code)
- **Interdits explicites** (CLAUDE.md) : `#0A0A0B`, `#08080A`, Outfit, Inter, system-ui seul

**Conflict scenarios concrets** :

| Scenario Kevin | Risque UI UX Pro Max |
|---|---|
| "Design new landing page for App Builder" | Auto-active → suggere glassmorphism palette rose/vert → **off Void Glass** |
| "Build dashboard widget" | Suggere Outfit font family → **interdit CLAUDE.md** |
| "Create hero section marketing" | Suggere #0A0A0B dark variant → **interdit CLAUDE.md** |
| "Improve button styling" | Suggere 5 palettes industry-matched → **100% off-brand** |

Le hook `validate-void-glass.sh` FOS bloquerait les Write directs, mais **les suggestions en conversation** seraient polluees. Kevin passerait son temps a corriger Claude qui suggere du off-brand.

**Si consultation-only via GitHub repo = HIGH** :
- Accessible CSV files dans src/ui-ux-pro-max/data/
- Piocher 1 palette one-shot et **l'adapter** a Void Glass (dark variant + ds-* tokens)
- Piocher 1 font pair one-shot et **verifier** si Figtree/JetBrains Mono compatible
- Aucune pollution auto-activation

### Risques / pieges

1. **Auto-activation pollue decisions DS** (point principal). Si Kevin installe global, toutes conversations "design" dans FOS recevront suggestions off-Void-Glass.
2. **Multi-framework suggest non-React** : FOS est React only. Skill peut suggerer Svelte/Vue/SwiftUI irrelevant.
3. **Python 3.x dep** : leger mais ajout dep FOS (verifier `python3 --version` sur machine Kevin).
4. **Industry matching** : skill matches "product type" → palette. FOS = **OS personnel Kevin**, pas un product type standard. Matching probablement faux.
5. **"Design System Generator" core = antithese FOS** : FOS a un DS **deja construit et fige** (Void Glass dark-only). Generator qui propose d'autres DS = contradiction philosophique.
6. **NextLevelBuilder org** a plusieurs projects (GoClaw.sh, ClaudeKit.cc, TOSE.sh) = ecosysteme commercial potentiel, risque pivot paid futur.
7. **Donations PayPal** = model supportable mais pas garantie maintenance long-terme.
8. **BM25 ranking** pour styles = heuristique simple. Quality recommendations variables.
